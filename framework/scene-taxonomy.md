# 场景分类体系说明

## 概述

本项目的工业场景分类基于国家标准《国民经济行业分类》(GB/T 4754)，结合人形机器人应用特点进行筛选和组织。

## 分类结构

### 行业维度（纵轴）

采用国标工业分类的三级编码体系：

```
大类代码 - 中类代码 - 小类代码

示例：36-361-3611 表示汽车整车制造
     33-00-00    表示金属制品业通用场景
```

当前覆盖 **41个工业大类**，包括：

| 代码 | 行业 | 场景数 |
|------|------|--------|
| 36 | 汽车制造业 | 34 |
| 39 | 计算机、通信和其他电子设备制造业 | 26 |
| 38 | 电气机械和器材制造业 | 33 |
| 35 | 专用设备制造业 | 32 |
| ... | ... | ... |

完整列表见 `data/scene-matrix-work.csv`

### 工序维度（横轴）

每个行业按其典型生产流程划分工序环节：

```
原料入库 → 预处理 → 加工成型 → 装配 → 检测 → 包装 → 成品出库
```

具体工序因行业而异，在场景数据的 `industry_chain` 字段中定义。

## 场景命名规范

采用三段式命名：

```
[动作] - [对象] - [约束条件]

示例：
- 搬运码垛-袋装水泥-50kg重载
- 旋拧紧固-仪表板螺栓-定扭矩
- 外观检测-注塑件-多面翻转
```

英文名称对应：

```
[Action] - [Object] - [Constraint]

示例：
- Palletizing-CementBags-50kgHeavyLoad
- BoltTightening-DashboardBolt-TorqueControlled
- VisualInspection-InjectionPart-MultiSideFlip
```

## 场景评估体系

每个场景从七个维度进行评估，评分范围1-5分：

| 维度 | 含义 | 分数越高表示 |
|------|------|--------------|
| 人工替代紧迫性 | 当前用人的痛点程度 | 替代需求越迫切 |
| 规模化潜力 | 可复制的工位数量 | 复制空间越大 |
| 超人类价值 | 机器人带来的增量价值 | 增量价值越多 |
| 操作复杂度 | 精度、力控要求 | 技术挑战越大 |
| 结构化程度 | 环境可预测性 | 环境越规整 |
| 容错空间 | 单次失败的后果 | 容错空间越大 |
| 节拍压力 | 产出速度要求 | 时间压力越小 |

### 综合价值分计算

```
综合价值分 = (人工替代紧迫性 + 规模化潜力 + 超人类价值) / 3
```

### 实施难度计算

```
实施难度 = (操作复杂度 + (5-结构化程度) + (5-容错空间) + (5-节拍压力)) / 4
```

## 场景数据字段

| 字段 | 说明 |
|------|------|
| scene_name_zh | 中文名称（三段式） |
| scene_name_en | 英文名称 |
| industry_code | 行业代码（国标） |
| industry_chain | 行业工序链 |
| process_step | 所属工序环节 |
| scene_description | 场景描述 |
| selection_rationale | 入选理由 |
| scores | 七维评分及依据 |
