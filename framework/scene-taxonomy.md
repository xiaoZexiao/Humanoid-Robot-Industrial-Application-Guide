# 场景分类矩阵说明

## 输出格式要求：
严格输出JSON数组，不输出任何JSON之外的文字。每个场景一个对象，字段定义如下：
json{
  "scene_name_zh": "",
  "scene_name_en": "",
  "industry_code": "",
  "industry_chain": "",
  "process_step": "",
  "scene_description": "",
  "selection_rationale": "",
  "scores": {
    "labor_urgency": { "score": 0, "basis": "" },
    "scale_potential": { "score": 0, "basis": "" },
    "superhuman_value": { "score": 0, "basis": "" },
    "operation_complexity": { "score": 0, "basis": "" },
    "structure_level": { "score": 0, "basis": "" },
    "fault_tolerance": { "score": 0, "basis": "" },
    "takt_pressure": { "score": 0, "basis": "" }
  }
}
各字段的生成规则：
scene_name_zh — 格式固定为"[动作]+[对象]+[约束条件]"，三段式命名。动作用动宾结构，对象写明操作的物料或设备，约束条件写该场景区别于同类场景的关键限定。例："搬运码垛-袋装水泥-50kg重载"、"旋拧紧固-仪表板M8螺栓-定扭矩"、"外观检测-注塑件-多面翻转"。禁止使用模糊名称如"智能装配"、"柔性操作"。
scene_name_en — 与中文名一一对应的英文翻译，格式为"[Action]-[Object]-[Constraint]"。例："Palletizing-CementBags-50kg"、"BoltTightening-DashboardM8-TorqueControlled"。单词间用驼峰，三段间用连字符。
industry_code — 格式为"大类编号-中类编号-小类编号"，编号取自国标工业分类（GB/T 4754）。若该场景适用于某大类下全部小类，中类和小类填"00"。例："36-361-3611"表示汽车整车制造，"33-00-00"表示金属制品业通用。
industry_chain — 该行业从原料进厂到成品出厂的主要工序链，各环节用"→"连接，限定6-12个环节。例："原料入库→裁切下料→冲压成型→焊接→表面处理→总装→检测→包装→成品出库"。同一industry_code下所有场景的industry_chain必须一致。
process_step — 从该场景的industry_chain中摘取一个环节，原文照抄，不得新造词。例：若industry_chain含"总装"，则此字段填"总装"。
scene_description — 严格五句话，每句话有固定职责：

第一句：该工位在产线上的物理位置和上下游关系。
第二句：当前该工位的执行方式（由[人工/半自动设备/XX]）和操作内容为[具体动作描述]。
第三句：核心操作的物理特征（例如：精度/力控/速度的范围），操作对象的物理特征（例如：物料名称、尺寸、重量、材质）。
第四句：该工位的典型班次和人员配置，单件操作耗时范围。
第五句：该工位容易出现的问题。

selection_rationale — 1-3句话，自由表述，依据筛选规则（不能照抄原文）

scores中每个维度：

score：整数1-5，严格按照评分标准表中对应档位的判定条件打分。
basis：一句话，≤30字，写出打该分数的直接事实依据或者工艺常识，不得重复评分标准的原文。例："年流动率约40%，需常年招聘"、"单件报废损失＜100元，可重试"。


一致性控制规则（写在prompt的系统指令部分）：

每个大类的输出结果末尾附一个汇总对象：
json{
  "summary": {
    "industry_major_code": "36",
    "industry_major_name": "汽车制造业",
    "total_scenes": 8,
    "industry_chain": "原料入库→冲压→焊装→涂装→总装→检测→包装→成品出库",
    "score_distribution": {
      "labor_urgency": {"1":0,"2":1,"3":3,"4":2,"5":2},
      "scale_potential": {"1":0,"2":0,"3":2,"4":4,"5":2}
    }
  }
}
这个汇总让你在41轮跑完后能快速扫一眼每个行业出了多少场景、评分分布是否合理，不用逐个打开看。

## scores中每个维度评价体系
1. 人工替代紧迫性（1-5）
判据：该工位当前用人做这件事有多痛。
分判定条件1招工容易，流动率低于行业平均，无安全/健康投诉2招工略有难度或存在单调性投诉，但不影响正常排产3存在明确痛点之一：疲劳导致质量波动需加检、需轮岗制增加冗余人力、年流动率超过30%4存在严重痛点之一：需穿戴重型防护使有效工时缩减超30%、招工周期超过3个月、已因人为失误造成过可量化损失5人类不可达或不宜到达：高温＞60℃/有毒/辐射/粉尘超标/极端洁净要求，或该工位近3年内发生过安全事故
2. 规模化潜力（1-5）
判据：该场景在单一企业内和跨企业间的可复制工位数量。
分判定条件1单企业内仅1-2个同类工位，且该场景仅存在于极少数特定企业2单企业内3-5个同类工位，或仅存在于单一细分行业3单企业内6-15个同类工位，且该场景在同行业内多数企业均存在4单企业内16-50个同类工位，或该场景可横向覆盖2-3个相关行业5单企业内超50个同类工位，或属于跨行业通用工序（如通用巡检、通用码垛），全国同类工位以万计
3. 超人类价值密度（1-5）
判据：机器人执行该任务时，除替代人工成本外还能产生多少种人类无法提供的增量价值。逐条比对以下七项，命中几项得几档。
七项增量价值清单：

V1 全时在线：可7×24连续作业，消除换班间隙和交接误差
V2 一致性输出：消除人类注意力衰减和个体差异导致的质量波动
V3 操作-检测合并：在执行操作的同时完成质量检测，取消独立检测环节
V4 群体数据共享：多台机器人的操作经验可聚合，新机部署时继承全部已有经验
V5 过程数据沉淀：每次操作自动生成可追溯的数字记录，满足质量追溯或合规要求
V6 超人感知：利用力矩/温度/气体等传感器获取人类无法感知的工艺信息
V7 失败数据价值：每次失败自动记录完整上下文，用于系统持续优化

分判定条件1命中0项，价值纯粹来自人工成本替代2命中1项3命中2-3项4命中4-5项5命中6-7项
4. 操作复杂度（1-5）
判据：该工位上人工执行的动作在精度、力控、接触状态转换三个维度上的要求。
分判定条件1纯自由空间运动，无接触精度要求。搬运、行走、巡视路线。2自由空间运动＋简单终端动作，接触但无精度约束。放置到大致位置、按大面积按钮。3受限接触操作，定位精度毫米级或有轻度力控要求，接触状态转换≤2次。拨拧大旋钮、简单插拔、抓放规则物体。4接触丰富操作，定位精度亚毫米级或需精细力控，接触状态转换3-5次。拧标准螺栓、插常规连接器、工具操作。5高度接触丰富操作，亚毫米级定位＋精细力控＋接触状态转换＞5次＋需实时异常判断。柔性材料操纵、精密插接、多步装配。
5. 结构化程度（1-5）
判据：该工位及其周边环境中，物体位置、人员活动、光照条件的可预测程度。分数越高越结构化。
分判定条件1布局不可预测，障碍物随机出现，大量人员和移动设备交叉。建筑工地、户外、家庭。2空间布局大致固定，但工件/物料位置不可预测，人员走动频繁。仓库拣选区、维修车间。3工位固定，物料位置有一定规律但存在波动（±10cm级），有限人机共存，光照有变化。一般制造车间。4工位固定，物料位置基本固定（波动±2cm内），偶有人员经过，光照稳定。半自动化产线。5一切预设：固定工位、固定工装、固定物料位姿、无人混行、光照恒定。洁净室、封闭自动化工位。
6. 容错空间（1-5）
判据：该工位上单次操作失败后的直接后果。分数越高越宽容。
分判定条件1单次失败导致：高价值物料报废（单件＞1万元）、或设备损伤、或安全事故风险、或产线停机＞30分钟2单次失败导致：单件物料报废（单件1千-1万元）或短暂停线（5-30分钟），损失可量化但可控3单次失败需人工介入矫正，耗时＜5分钟，不报废物料不停线4单次失败可立即重试，有轻微时间成本（重试耗时＜30秒），不造成任何物料损伤5失败完全无后果，可下轮补上或自动跳过。巡检漏读、非关键数据采集遗漏。
7. 节拍压力（1-5）
判据：该工位的产出速度要求对执行者速度的约束程度。分数越高越宽松。
分判定条件1瓶颈工位，节拍≤15秒/件，工位物理空间不允许并行部署第二台设备2节拍要求严格（15-30秒/件），空间仅允许并行1台，速差补偿后勉强达标3节拍要求中等（30-120秒/件），或空间允许并行部署2台以上来补偿速差4有节拍但该工位非瓶颈，余量充足，即使执行速度为人工的一半也不拖累整线5无节拍要求，任务无固定产出速度限制。巡检、非产线任务、离线质检。
